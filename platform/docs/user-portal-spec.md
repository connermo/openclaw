# OpenClaw 用户入口 - 前端界面规格说明

> 本文档描述 OpenClaw 管理平台的**用户入口**（`/app/*`），面向终端用户管理个人 AI 助手（1 用户 = 1 Gateway = 1 AI Agent）。技术栈：React 19、TypeScript、TanStack Router + Query、shadcn/ui + Tailwind CSS、Recharts、Vite。

---

## 全局布局

- **顶部导航栏**：Logo（OpenClaw）、全局搜索（搜索会话/技能/设置，Cmd/Ctrl+K 唤起命令面板）、通知铃铛图标（带未读数量角标）、用户头像下拉菜单（个人资料、设置、退出登录）。
- **左侧边栏**：导航菜单，带图标，包含各板块入口 — 首页、对话、我的助手、技能市场。当前激活项高亮显示。小屏幕可折叠。
- **主内容区**：侧边栏右侧，渲染各页面内容。响应式布局，最大宽度容器，适当内边距。
- **配色方案**：简洁现代的 SaaS 风格。默认浅色主题，支持深色模式切换。主题强调色可自定义。

## 1. 首页（Dashboard） — `/app`

登录后的着陆页，一屏概览回答："我的助手在做什么？" 和 "最近发生了什么？"

### 1.1 助手状态卡片

- **位置**：页面顶部，全宽卡片。
- **内容**：
  - 助手头像（可自定义）+ 助手名称（如"我的助手"）
  - 状态指示灯：绿色圆点 = 在线，黄色 = 异常，红色 = 离线，灰色 = 未配置
  - 当前 LLM 模型名称（如 "Claude Sonnet 4"）
  - 运行时长（如 "已运行 3 天 12 小时"）
  - 已连接渠道徽章（如 Telegram、Slack、微信图标及状态）
- **操作按钮**："开始对话" 按钮（跳转至对话页），"配置助手" 链接（跳转至我的助手页）。

### 1.2 今日摘要

- **位置**：状态卡片下方，一行 3-4 个统计卡片。
- **卡片内容**（每个包含图标、数字和标签）：
  - 今日处理消息数（聊天图标）
  - 今日执行任务数（对勾图标）
  - 今日消耗 Token 数（闪电图标）
  - 活跃会话数（对话框图标）
- **样式**：圆角卡片，浅色背景，大号粗体数字，下方小号标签文字。

### 1.3 最近对话

- **位置**：摘要统计下方左列（桌面端双栏布局）。
- **内容**：最近 5 条对话摘要列表。
  - 每条：渠道图标、对话首句（截断显示）、相对时间戳（如 "2 小时前"）、消息数量角标。
- **交互**：点击任意条目跳转至对话页查看完整对话。
- **空状态**：插画 + "暂无对话，开始和你的助手聊天吧！"

### 1.4 任务时间线

- **位置**：最近对话右侧列。
- **内容**：垂直时间线，展示近期事件：
  - 定时任务执行（时钟图标、任务名称、结果：成功/失败、时间）
  - 子代理任务完成（机器人图标、任务描述、状态、时间）
  - 技能安装/更新（拼图图标、技能名称、操作类型、时间）
- **交互**：点击事件查看详情（内联展开或跳转）。
- **空状态**："暂无近期任务。"

### 1.5 通知区域

- **位置**：双栏区域下方，或通过顶部导航栏铃铛图标访问。
- **内容**：告警卡片，包括：
  - 助手离线 / 恢复在线
  - 定时任务执行失败
  - 用量配额告警（接近上限）
  - 组织发布了新技能
- **样式**：可关闭的卡片，按严重程度着色（红色 = 错误，黄色 = 警告，蓝色 = 信息）。

## 2. 对话（Chat） — `/app/chat`

核心对话界面，用户与 AI 助手实时交互。

### 2.1 会话列表（左侧面板）

- **位置**：左侧面板（可拖拽调整宽度，默认约 300px）。
- **头部**："对话" 标题 + "新建对话" 按钮（+ 图标）。
- **搜索与筛选**：搜索框（按关键词过滤）。筛选标签或下拉菜单：全部、按渠道（Telegram、Slack、Web 等）、按时间范围。
- **会话条目**：列表形式，每条显示：
  - 渠道图标（对话来源）
  - 会话标题（自动生成或首条消息预览，截断为 1 行）
  - 最后消息时间戳（相对时间）
  - 未读指示圆点（如有新消息）
  - 消息数量角标
- **排序**：默认按最近排序，可选按渠道分组。
- **激活状态**：选中的会话以强调色背景高亮。
- **空状态**："暂无对话，点击'新建对话'开始。"

### 2.2 聊天界面（主区域）

- **位置**：中央/右侧主内容区。
- **头部栏**：会话标题（可编辑）、渠道徽章、会话时长、"..." 菜单（删除会话、导出记录）。
- **消息列表**：可滚动区域，显示消息：
  - **用户消息**：右对齐，强调色气泡，头像在右侧。
  - **助手消息**：左对齐，中性色气泡，助手头像在左侧。
    - 支持 Markdown 渲染（代码块带语法高亮、表格、列表、链接）。
    - 工具调用指示器：可折叠区域，显示工具名称、输入参数、输出结果（代码块样式），图标标识工具类型。
    - 思考过程：可折叠的 "思考中..." 区域，展开显示推理文本（暗色/斜体样式）。
    - 流式输出：消息逐字出现，显示打字指示器（闪烁光标或动态圆点）。
  - **系统消息**：居中显示，弱化文字（如 "会话已开始"、"渠道已切换"）。
  - 时间戳在悬停时显示，或在时间间隔处显示（如 "今天 14:30"）。
- **输入区域**：固定在底部。
  - 多行文本输入框（自动扩展高度，最大约 6 行后滚动）。
  - 发送按钮（主题色，箭头图标）。快捷键：Enter 发送，Shift+Enter 换行。
  - 附件按钮（回形针图标），支持文件/图片上传。
  - 流式输出时显示"停止"按钮（替换发送按钮，方块图标）。
- **空状态**（未选择会话）：居中插画 + "选择一个对话或开始新对话。"

### 2.3 新建对话

- 点击"新建对话"按钮触发。
- 直接打开新的空白聊天会话（无需弹窗），输入框自动获得焦点。
- 可选：如果用户连接了多个渠道，显示渠道选择下拉菜单（默认 Web）。

### 2.4 会话详情 / 记录查看

- 可从会话列表或首页最近对话进入。
- 布局与聊天界面相同，但对已结束会话为只读模式。
- 附加功能：
  - 完整记录，所有工具调用默认展开。
  - 长会话可使用"跳转到"导航（锚点链接到工具调用、关键消息）。
  - 导出按钮：下载为 Markdown 或 JSON 格式。

## 3. 我的助手（配置中心） — `/app/assistant`

用户个人 AI 助手的配置管理中心。以设置页面形式组织，包含多个分区。

### 3.1 身份设置 — `/app/assistant/identity`

- **助手名称**：文本输入框（如 "Aria"、"我的助手"）。
- **头像**：上传自定义图片或从预设头像中选择，圆形预览。
- **标识 Emoji**：Emoji 选择器，选择一个快速标识用的 Emoji（显示在状态卡片、通知中）。
- **主题色**：取色器或预设色板（强调色用于助手相关 UI 元素）。
- **保存按钮**：保存更改，成功后显示 Toast 提示。

### 3.2 模型选择 — `/app/assistant/model`

- **当前模型**：醒目展示，带供应商 Logo + 模型名称。
- **模型选择器**：下拉菜单或卡片网格，按供应商分组：
  - 供应商 Logo（Anthropic、OpenAI、Google 等）
  - 模型名称、描述、能力标签（如 "快速"、"高级推理"、"视觉理解"）
  - 成本指示（相对标识：$、$$、$$$）
- **供应商 API 密钥**：如用户自带密钥，提供输入框及显示/隐藏切换，加密状态指示。
- **高级设置**：可折叠区域，包含 Temperature 滑块、最大 Token 输入框、系统提示词覆盖。

### 3.3 渠道绑定 — `/app/assistant/channels`

- **已连接渠道列表**：每个渠道一张卡片：
  - 渠道图标 + 名称（Telegram、Slack、Discord、微信、Signal、iMessage 等）
  - 连接状态：绿色 = 已连接，红色 = 已断开，灰色 = 未配置
  - 账号/机器人信息（如 "Telegram 上的 @my_bot"）
  - 最后活跃时间
  - 操作："重新连接" 按钮（断开时显示）、"断开连接" 按钮（带确认弹窗）、"配置" 链接
- **添加渠道**：列表末尾 "+ 添加渠道" 按钮。打开弹窗/页面，展示可用渠道类型及配置说明。
- **空状态**："暂无已连接渠道。添加渠道让你的助手跨平台沟通！"

### 3.4 工具与权限 — `/app/assistant/tools`

- **工具列表**：表格或卡片列表展示所有可用工具：
  - 工具图标 + 名称 + 描述（1 行）
  - 分类标签（如 "文件系统"、"网络"、"代码执行"、"通信"）
  - 开关切换：启用/禁用
  - 权限级别下拉菜单："自动批准"、"使用前询问"、"禁用"
- **批量操作**："全部启用"、"全部禁用" 按钮。
- **搜索/筛选**：按工具名称搜索，按分类筛选。
- **审批设置**：顶部全局默认权限级别选择器。

### 3.5 定时任务 — `/app/assistant/cron`

- **任务列表**：表格，包含以下列：
  - 任务名称（可内联编辑）
  - 调度表达式（人类可读格式，如 "每天上午 9:00"，悬停显示原始 cron 表达式）
  - 提示词/命令（截断显示，可展开）
  - 状态：启用/暂停 开关
  - 上次执行：时间戳 + 结果（成功/失败 徽章）
  - 下次执行：时间戳
  - 操作：编辑、立即执行、删除
- **创建任务**："+ 新建任务" 按钮。打开表单：
  - 任务名称输入框
  - 调度设置：预设选项（每天、每周、每小时、自定义）+ cron 表达式输入框及人类可读预览
  - 提示词：多行文本框（发送给助手的消息/命令）
  - 保存/取消 按钮
- **编辑任务**：与创建相同的表单，预填充数据。
- **执行历史**：每个任务可展开查看，显示最近 N 次执行记录，包含时间戳、耗时、结果、输出预览。

### 3.6 个性化文件 — `/app/assistant/workspace`

- **文件列表**：左侧文件列表：
  - `SOUL.md` — 助手人格与行为准则
  - `IDENTITY.md` — 身份与上下文信息
  - 用户创建的自定义文件
- **编辑器**：主区域为代码/Markdown 编辑器：
  - Markdown 语法高亮
  - 行号显示
  - 自动保存指示器（或显式保存按钮，支持 Cmd/Ctrl+S 快捷键）
  - 预览切换：并排模式或标签页切换编辑/预览视图
- **创建文件**："+ 新建文件" 按钮，弹出文件名输入框。
- **删除文件**：文件右键菜单或 "..." 菜单中提供，带确认弹窗。

## 4. 技能市场（Skill Store） — `/app/skills`

类似应用商店的体验，用于发现和管理 AI 技能/插件。

### 4.1 技能市场首页 — `/app/skills`

- **头部**："技能市场" 标题 + 搜索栏（醒目居中）。
- **推荐技能区**：水平轮播或网格展示组织推荐的技能。每张卡片：
  - 技能图标（彩色、有辨识度）
  - 技能名称
  - 简短描述（1-2 行）
  - 作者/发布者名称
  - 安装数量或评分
  - "安装" 按钮（已安装则显示 "已安装" 徽章）
- **分类浏览**：分类卡片网格或水平标签栏：
  - 分类：效率工具、通信协作、数据分析、开发工具、内容创作、自动化 等
  - 每个分类卡片：图标、名称、技能数量
- **热门技能**：按安装数量排序的技能网格。
- **最新上架**：按上架时间排序的技能网格。

### 4.2 分类页面 — `/app/skills/category/:categoryId`

- **头部**：分类名称 + 描述。
- **技能网格**：响应式网格（桌面 3-4 列，平板 2 列，手机 1 列）。
- **排序**：按热度、最新、名称（A-Z）排序。
- **分页**：无限滚动或"加载更多"按钮。

### 4.3 技能详情 — `/app/skills/:skillId`

- **头部区域**：
  - 大号技能图标
  - 技能名称（h1）
  - 作者/发布者（带链接）
  - 版本号
  - 安装数量
  - "安装" / "卸载" 按钮（主要操作）
- **描述**：完整 Markdown 描述，可包含截图/示例。
- **使用说明**：代码示例或使用说明，展示技能如何与助手配合工作。
- **所需权限**：技能需要的权限列表（如 "文件访问"、"网络访问"）。
- **更新日志**：可折叠的版本历史。
- **相关技能**：底部水平展示相似技能列表。

### 4.4 我的技能 — `/app/skills/installed`

- **头部**："我的技能" 标题。
- **已安装技能列表**：表格或卡片列表：
  - 技能图标 + 名称 + 版本号
  - 状态：启用/禁用 开关
  - 安装日期
  - 自动更新开关
  - 操作："配置"（如技能有设置项）、"卸载" 按钮
- **空状态**："尚未安装任何技能。去技能市场探索，增强你的助手能力！"
- **可用更新**：有更新的技能显示更新角标。如有多个更新，顶部显示"全部更新"按钮。

## 5. 个人设置 — `/app/settings`

通过顶部导航栏用户头像下拉菜单访问。

### 5.1 个人资料

- 显示名称、邮箱（SSO 登录时只读）、头像上传。
- 所属部门（只读，由管理员设置）。
- 已关联的认证方式信息（如 "通过 Google 登录"）。

### 5.2 通知偏好

- 每种通知类型的开关设置（邮件、应用内、推送）：
  - 助手状态变更
  - 定时任务失败
  - 用量告警
  - 组织发布新技能

### 5.3 用量统计

- **用量仪表盘**：个人用量统计面板。
  - Token 消耗趋势图（按日/周/月，折线图）
  - 按模型的成本分布（柱状图或饼图）
  - 会话数量趋势
  - 当前计费周期使用量与配额对比（进度条）
- **用量明细表**：按日期、模型、会话、Token 数、成本的详细分解表格。

### 5.4 API 密钥

- 个人编程访问密钥管理。
- 密钥列表：名称、创建日期、最后使用时间、权限范围、过期时间。
- 创建密钥：名称输入、权限范围选择（复选框）、过期时间选择器。
- 吊销密钥：带确认弹窗。
- 密钥仅在创建时显示一次（带复制按钮 + 警告提示）。

## 6. 实时推送与 WebSocket 行为

以下实时更新通过 WebSocket 反映到界面：

| 事件 | 界面更新 |
|------|---------|
| `assistant.status.changed` | 首页状态卡片即时更新（状态圆点、状态文字），弹出 Toast 通知。 |
| `chat.message` | 新消息实时流入当前激活的聊天会话（逐字输出）。 |
| `cron.executed` | 首页任务时间线更新。定时任务列表"上次执行"更新。失败时弹出 Toast。 |
| `skill.update` | 技能市场导航项显示角标。"我的技能"显示可用更新。 |

## 7. 响应式设计说明

| 断点 | 布局 |
|------|------|
| 桌面端（>1024px） | 侧边栏 + 主内容区并排。对话页：会话列表 + 聊天区域并排。 |
| 平板端（768-1024px） | 侧边栏可折叠（汉堡菜单）。对话页：会话列表作为浮层/抽屉。 |
| 移动端（<768px） | 底部标签导航替代侧边栏。对话页：全屏会话列表或聊天区域（在两者间导航切换）。 |

## 8. 核心 UI 组件（shadcn/ui）

- **卡片**：状态卡片、统计卡片、技能卡片（使用 `Card`、`CardHeader`、`CardContent`）。
- **数据表格**：工具列表、定时任务、用量表格（使用 TanStack Table，支持排序、筛选、分页）。
- **表单**：所有配置表单（使用 `Form`、`Input`、`Select`、`Switch`、`Textarea`）。
- **弹窗**：确认弹窗、新建任务、渠道配置（使用 `Dialog`、`AlertDialog`）。
- **Toast 通知**：成功/错误/信息提示（使用 `Sonner` 或 shadcn toast）。
- **标签页**：页面内分区导航（使用 `Tabs`、`TabsList`、`TabsTrigger`、`TabsContent`）。
- **命令面板**：全局搜索（使用 `Command` 组件，Cmd/Ctrl+K 唤起）。
- **骨架屏**：数据加载时显示（所有异步内容使用 `Skeleton` 组件）。
- **徽章**：状态指示器、渠道徽章、通知数量角标（使用 `Badge`）。
- **头像**：用户头像、助手头像（带状态圆点叠加，使用 `Avatar`）。
- **图表**：Recharts 绘制用量/成本图表（LineChart、BarChart、PieChart）。
